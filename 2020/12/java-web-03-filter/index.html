Java web 之 Filter

Author: 张潇 Date: December 28, 2020  15:39:27

Filter 在整个数据请求中的位置处于 Web 服务器和 Web 资源中间，所以可以起到 2 部分的作用

  • 在访问资源之前，对请求进行验证、处理[1]和过滤。
  • 在访问资源之后，对于目标资源进行一些特殊处理

Java 中 Filter 的实现

在 Java 中，只要是实现了 Filter 接口的类就可以成为一个过滤器

其中，init() 和 destroy() 的用法类似于 在Servlet 中的用法，只有在 Web 服务器加载、销毁该 Filter 的时候的时候被执行一次。

需要特殊注意的是 doFilter() 方法，更需要特殊注意的是 FilterChain 参数

可以看到 FilterChain 也是一个接口，并且还有一个同名的方法 doFilter()，这个该如何理解呢？这就涉及到了 Filter 的管理。

Filter 的管理

首先来说，一个 Web 应用肯定不止一个过滤器，那么如何管理这一些 Filter 呢？

类似于自来水过滤的设计原理，我们将一个一个过滤器串行的链接起来，这就是 FilterChain。

这就延伸出来两个问题

  • 过滤器执行的顺序该如何设置？
  • 不同 Filter 是怎么连结起来的？

Filter 的执行顺序

在将执行顺序之间，我们必须先说一下 Filter 的部署方式

Filter 两种部署方式

通过 web.xml 进行配置

用于给 Filter 指定一个名字，该元素不能为空
用于指定该 Filter 对应实现类的全限定名

设置 Filter 所拦截的请求路径

通过注解配置

“/*” 表示访问任何资源都要经过 FilterName
过滤器

执行顺序

过滤器之间的执行顺序看在 web.xml 文件中 mapping 的先后顺序的，如果放在前面就先执行，放在后面就后执行！
如果是通过注解的方式配置，就比较 urlPatterns 的字符串优先级。

不同 Filter 的连结

不同 Filter 的连结是通过调用 FilterChain 的 doFilter 方法实现的。
